## 3. Shipped support tier (authoritative)

Source of truth: `policy/host-support-matrix.tsv`. Do not restate a broader
claim than these rows.

- **Supported, launch allowed (Apple Silicon macOS only):**
  - `macos/arm64/local_vm/colima/strict` — reviewed strict operator-host path.
  - `macos/arm64/local_compat/docker-desktop/compat` — lower assurance than
    strict Colima.
- **Preview-only, launch blocked:**
  - `macos/arm64/remote_vm/aws-ec2-ssm/compat`
  - `macos/arm64/remote_vm/gcp-vm/compat`
  - `macos/arm64/local_vm/apple-container/per-session-vm` — the C1 evaluation
    result: GO on the evaluation, but support-invisible and launch-blocked for
    1.0. Only this preview backend requires macOS 26 (`RequireMacOS26()`);
    supported launch on Apple Silicon macOS does **not** require macOS 26 — both
    strict Colima (the default) and docker-desktop compat are supported/allowed
    with no version floor in `policy/host-support-matrix.tsv`, so docker-desktop
    compat is a supported launch option for hosts below macOS 26.
- **Validation-host-only (CI, not an operator launch host):**
  - `linux/amd64/local_vm/colima/strict` and the two `linux/amd64` remote-VM
    deterministic-preview rows.
- **Unsupported, launch blocked:** all Linux `arm64`, all Windows, and Linux
  `amd64` Docker Desktop rows.

**Provider tier (source: `docs/provider-matrix.md`):** Tier 1 supported adapters
today are **Codex, Claude Code, GitHub Copilot CLI, Gemini**. **Google
Antigravity CLI is a fail-closed scaffold — not current support** (queued
Tier 1 track). Gemini's adapter stays supported for the auth inputs Google keeps
serving (licensed Code Assist or a paid Gemini API key, `gcloud_adc`
supplemental); Google — not Workcell — retires personal-account OAuth login on
2026-06-18.

## 5. Honest non-claims / known gaps (do not overstate)

Grounded in `ROADMAP.md` Non-Goals plus verification of what is on `main`:

- **No Linux, Windows, Linux `arm64`, or Raspberry Pi operator-host support.**
  Linux `amd64` is a trusted validation host only.
- **Apple `container` is preview-only/launch-blocked** for 1.0; promotion is
  deferred post-1.0 pending the B6 certification lane.
- **Antigravity is fail-closed scaffold**, not a support claim. 1.0 may ship
  with the scaffold plus a recorded scope decision (ROADMAP criterion 4).
- **A5 signed audit records are not yet on `main`** (in final review).
- **B2 dual-control release approval:** not found on `main`.
- **B6 real-boundary certification lane:** not implemented. Criterion 6 requires
  it in place; deferring it requires a recorded amendment to criterion 6, not a
  free disposition — a human/funding decision. The live Colima/Apple-Silicon
  boundary is not exercised in hosted CI today (only a Docker smoke lane on a
  `ubuntu-latest` runner is). See `docs/b6-disposition-options-draft.md`.
- **C2 latency numbers:** the benchmark methodology/harness ship
  (`docs/session-startup-benchmarks.md`, `scripts/bench/`) but published numbers
  are pending live capture — verify before any latency claim.
- **C3 parallel sessions:** 1.0 delivers container-level isolation on the strict
  path; full worktree-per-agent isolation should be verified against the
  criterion before sign-off.
- **G1 freeze (part 2):** the contract *inventory* exists; the *freeze* +
  deprecation policy is the v1.0-rc G1 step and is intentionally LAST (after A5
  and the contract surface settle). Do not run the freeze before then.
- **D3 (complete) / D6:** see the D3 and D6 status notes in
  `docs/improvement-tracks-implementation-plan.md` — both have literal-exit-gate
  scope decisions pending for this review.
